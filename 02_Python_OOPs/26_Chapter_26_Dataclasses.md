# Python OOP: Chapter 26 Dataclasses

Dataclasses are one of the most powerful features introduced in Python 3.7 to simplify the creation of classes that mainly store data.

Before dataclasses, developers had to write repetitive code for:

* `__init__()`
* `__repr__()`
* `__eq__()`
* Comparison methods
* Default value handling

The `dataclasses` module automates this process and allows developers to define data-centric classes with very little code while still maintaining full flexibility.

This article explains in detail:

* `dataclasses` module
* `@dataclass` decorator
* Automatic method generation
* Default values
* Immutable dataclasses

with practical examples and deep explanations.

---

# 1. The `dataclasses` Module

## Concept

The `dataclasses` module provides a decorator and utilities that automatically generate common methods for classes whose primary purpose is storing data.

Instead of writing verbose classes like this:

```python
class User:
    def __init__(self, name, age):
        self.name = name
        self.age = age

    def __repr__(self):
        return f"User(name={self.name}, age={self.age})"

    def __eq__(self, other):
        return (
            self.name == other.name and
            self.age == other.age
        )
```

We can write:

```python
from dataclasses import dataclass

@dataclass
class User:
    name: str
    age: int
```

Python automatically generates:

* `__init__()`
* `__repr__()`
* `__eq__()`

This dramatically reduces boilerplate code.

---

## Why Dataclasses Were Introduced

Before dataclasses, developers commonly used:

* `namedtuple`
* Simple classes
* `attrs` library

However, these approaches had limitations such as:

* Limited flexibility
* Complex syntax
* Lack of extensibility

Dataclasses combine the simplicity of named tuples with the flexibility of regular classes.

# 2. The `@dataclass` Decorator

## Concept

The `@dataclass` decorator modifies a class at creation time to automatically generate methods based on class field definitions.

Example:

```python
from dataclasses import dataclass

@dataclass
class Product:
    name: str
    price: float
```

When Python interprets this class, the decorator automatically creates methods.

Equivalent expanded class:

```python
class Product:
    def __init__(self, name: str, price: float):
        self.name = name
        self.price = price

    def __repr__(self):
        return f"Product(name={self.name}, price={self.price})"

    def __eq__(self, other):
        return (
            self.name == other.name and
            self.price == other.price
        )
```

---

## Creating Instances

Example:

```python
p = Product("Laptop", 80000)

print(p)
```

Output:

```text
Product(name='Laptop', price=80000)
```

Notice the automatically generated string representation.

---

# 3. Fields in Dataclasses

Fields are defined using type annotations.

Example:

```python
@dataclass
class Student:
    name: str
    age: int
    grade: float
```

Each annotated attribute becomes a dataclass field.

Fields are stored internally in:

```python
__dataclass_fields__
```

Example:

```python
print(Student.__dataclass_fields__)
```

This contains metadata about each field.

---

# 4. Automatic Method Generation

The `@dataclass` decorator can generate several methods automatically.

### Generated by Default

* `__init__()`
* `__repr__()`
* `__eq__()`

### Optionally Generated

* `__lt__()`
* `__le__()`
* `__gt__()`
* `__ge__()`
* `__hash__()`

These methods allow dataclasses to behave like structured data objects.

---

## 4.1 Automatic `__init__()`

Example:

```python
@dataclass
class Point:
    x: int
    y: int

p = Point(10, 20)

print(p.x)
```

Output:

```text
10
```

The constructor is automatically generated.

---

## 4.2 Automatic `__repr__()`

Dataclasses automatically generate a readable representation.

Example:

```python
p = Point(3, 4)

print(p)
```

Output:

```text
Point(x=3, y=4)
```

This is extremely useful for debugging.

---

## 4.3 Automatic Equality Comparison

Two dataclass instances can be compared.

Example:

```python
p1 = Point(1, 2)
p2 = Point(1, 2)

print(p1 == p2)
```

Output:

```text
True
```

The equality method compares all fields.

Equivalent logic:

```python
p1.x == p2.x and p1.y == p2.y
```

---

## 4.4 Ordering Support

Dataclasses can automatically support comparisons.

Example:

```python
@dataclass(order=True)
class Item:
    price: int
```

Usage:

```python
i1 = Item(10)
i2 = Item(20)

print(i1 < i2)
```

Output:

```text
True
```

Python automatically generates:

* `__lt__()`
* `__le__()`
* `__gt__()`
* `__ge__()`

# 5. Default Values

Fields can have default values.

Example:

```python
@dataclass
class User:
    name: str
    age: int = 18
```

Usage:

```python
u = User("Alice")

print(u.age)
```

Output:

```text
18
```

Default values behave exactly like normal function defaults.

---

## Default Ordering Rule

Fields with defaults must come after fields without defaults.

Incorrect example:

```python
@dataclass
class User:
    age: int = 18
    name: str
```

This raises an error because Python requires non-default arguments first.

---

# 6. Mutable Default Values

Mutable defaults must be handled carefully.

Incorrect example:

```python
@dataclass
class User:
    hobbies: list = []
```

### Problem

All instances share the same list.

---

## Correct Solution: `default_factory`

```python
from dataclasses import dataclass, field

@dataclass
class User:
    hobbies: list = field(default_factory=list)
```

Now each instance gets its own list.

Example:

```python
u1 = User()
u2 = User()

u1.hobbies.append("reading")

print(u2.hobbies)
```

Output:

```text
[]
```

Lists are no longer shared.

---

# 7. The `field()` Function

`field()` allows customization of dataclass fields.

Example import:

```python
from dataclasses import field
```

Supported options include:

* `default`
* `default_factory`
* `init`
* `repr`
* `compare`
* `metadata`

---

## Example

```python
@dataclass
class User:
    name: str
    password: str = field(repr=False)
```

Now the password will not appear in printed output.

Usage:

```python
u = User("Alice", "secret")

print(u)
```

Output:

```text
User(name='Alice')
```

---

# 8. Immutable Dataclasses

Dataclasses can be made immutable (read-only).

This is done using:

```python
frozen=True
```

Example:

```python
@dataclass(frozen=True)
class Point:
    x: int
    y: int
```

Usage:

```python
p = Point(1, 2)

p.x = 10
```

Error:

```text
FrozenInstanceError
```

Fields cannot be modified.

---

## Why Immutable Dataclasses Are Useful

Immutable objects provide:

* Thread safety
* Predictability
* Functional programming style
* Safe hashing

They behave similarly to tuples.

---

# 9. Dataclasses and Hashing

Frozen dataclasses automatically support hashing.

Example:

```python
@dataclass(frozen=True)
class Point:
    x: int
    y: int
```

Usage:

```python
points = {
    Point(1, 2),
    Point(3, 4)
}

print(points)
```

Because objects are immutable, they can safely be used in:

* Sets
* Dictionary keys

---

# 10. Post Initialization

Dataclasses support a special method:

```python
__post_init__()
```

It runs after the generated `__init__()`.

Example:

```python
@dataclass
class Rectangle:
    width: int
    height: int
    area: int = 0

    def __post_init__(self):
        self.area = self.width * self.height
```

Usage:

```python
r = Rectangle(5, 4)

print(r.area)
```

Output:

```text
20
```

This allows derived field calculations.

---

# 11. Dataclasses with Slots (Memory Optimization)

Dataclasses can be combined with slots.

Example:

```python
@dataclass(slots=True)
class Point:
    x: int
    y: int
```

Benefits:

* Reduced memory usage
* Faster attribute access

This is useful when creating millions of objects.

# 12. Dataclass Utility Functions

The module also provides helper functions.

---

## `asdict()`

Converts a dataclass instance into a dictionary.

Example:

```python
from dataclasses import asdict

p = Point(1, 2)

print(asdict(p))
```

Output:

```python
{'x': 1, 'y': 2}
```

---

## `astuple()`

Converts a dataclass instance into a tuple.

Example:

```python
from dataclasses import astuple

print(astuple(p))
```

Output:

```python
(1, 2)
```
